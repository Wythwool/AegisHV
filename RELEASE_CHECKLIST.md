# Release Checklist

This checklist is for a host-side sensor release. It is not a type-1 or full VMI release checklist.

## Build

- `cargo metadata --locked --format-version 1`
- `cargo fmt --all -- --check`
- `cargo clippy --locked --all-targets --all-features -- -D warnings`
- `cargo test --locked --all --all-features`
- `scripts/smoke-replay.sh`
- `scripts/check-doc-links.sh`

## Artifacts

- Release archive built by `scripts/package-release.sh`.
- Checksums written for every archive.
- SBOM generated from packaged inputs.
- Sigstore bundle generated by the release workflow when signing is enabled.
- Release notes list exact supported runtime paths and unsupported backend paths.

## Manual Review

- Hardware matrix reviewed for claim drift.
- Performance result files attached only when raw outputs are present.
- Security checklist reviewed.
- Changelog updated.
- README checked against `docs/STATUS.md`.

## Stop Conditions

- Any failing locked test.
- Any missing release artifact.
- Any unsupported backend claim in README, changelog, docs, package metadata, or release notes.
- Any benchmark number without raw output and host metadata.
