# Attestation Notes

This tree contains release provenance documentation and a small build-info surface. It does not implement hardware remote attestation, measured boot, TPM quote verification, or type-1 launch measurement.

## Local Build Info

`aegishv version --json` reports:

- package version;
- target architecture;
- target operating system;
- compile-time Git revision when supplied through `AEGISHV_GIT_REV`.

The Git revision field is informational. It is not a signature and it does not prove the binary came from a trusted build.

## Startup Hash Events

`src/tamper.rs` includes bounded digest helpers for local file-change evidence. The digest label is explicit and intentionally not described as cryptographic authentication. Use external signing, filesystem integrity controls, or a platform measurement chain when integrity proof is required.

## Release Evidence

Release notes should keep these items together:

- source commit;
- locked dependency graph;
- build target;
- release archive checksum;
- SBOM;
- Sigstore bundle when generated by the release workflow;
- operator test log.

## Boundary

Hardware attestation and measured type-1 launch evidence are not implemented. Any deployment requiring those properties must add a real measurement path, verifier policy, replayable logs, and negative tests before making those claims.
