# Architecture Decision Records

This directory records architecture decisions for AegisHV.

ADRs are for decisions that change a long-lived boundary, operator contract, event contract, security posture, dependency policy, backend interface, or release rule. Small implementation details should stay in code review.

Each ADR uses these sections:

- Status
- Context
- Decision
- Consequences
- Test Impact
